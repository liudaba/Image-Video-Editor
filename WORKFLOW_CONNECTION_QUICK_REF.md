# 视频生成任务 - 环节衔接快速参考

## ✅ 已修复的关键问题

### 🔴 P0严重问题（已全部修复）

#### 1. 分镜不再重复生成
- **修复前**: 即使有shots_data.json，也会重新生成分镜，浪费5-10分钟
- **修复后**: 检测到已有分镜脚本时直接使用，跳过生成步骤
- **验证方法**: 运行"跑图生成视频"，观察日志是否显示"✅ 检测到已存在的分镜脚本文件"

#### 2. 不会误删已有图片
- **修复前**: skip_clear=False导致每次都会清除所有图片
- **修复后**: skip_clear=True保护已有图片
- **验证方法**: 手动放几张测试图片，运行任务后检查图片是否还在

## 🔄 完整流程说明

### 正常流程（有分镜脚本）
```
1. 检查音频文件 ✅
2. 启动渲染线程
   ├─ 阶段1: 准备分镜数据 (0-30%)
   │   ├─ 检测到shots_data.json
   │   ├─ 加载分镜数据 [✅ 复用，不重新生成]
   │   └─ 验证分镜完整性
   │
   ├─ 阶段2: 生成图像 (30-70%)
   │   ├─ 检查图片是否存在
   │   ├─ 如果缺失 → 调用generate_images()
   │   └─ 记录生成耗时
   │
   └─ 阶段3: 视频合成 (70-100%)
       ├─ 创建视频片段
       ├─ 添加音频
       └─ 渲染输出
```

**总耗时**: 约2-5分钟（取决于图片数量）

---

### 首次运行流程（无分镜脚本）
```
1. 检查音频文件 ✅
2. 启动渲染线程
   ├─ 阶段1: 准备分镜数据 (0-30%)
   │   ├─ 未检测到shots_data.json
   │   ├─ 转录音频 (Whisper) [⏱️ 1-2分钟]
   │   ├─ 分析内容 (Ollama) [⏱️ 2-3分钟]
   │   └─ 生成分镜脚本
   │
   ├─ 阶段2: 生成图像 (30-70%)
   │   ├─ 检查图片是否存在
   │   ├─ 全部缺失 → 调用generate_images() [⏱️ 5-10分钟]
   │   └─ 记录生成耗时
   │
   └─ 阶段3: 视频合成 (70-100%)
       ├─ 创建视频片段
       ├─ 添加音频
       └─ 渲染输出 [⏱️ 1-2分钟]
```

**总耗时**: 约10-20分钟（首次运行）

## ⚠️ 常见问题

### Q1: 为什么还是重新生成了分镜？
**A**: 检查以下几点：
1. 输出文件夹是否有`shots_data.json`文件
2. 文件是否是有效的JSON格式
3. 查看日志中是否有"❌ 加载分镜数据失败"的提示
4. 如果加载失败，会重新生成，这是正常的

### Q2: 图片被删除了怎么办？
**A**: 
1. 修复后不会再发生这个问题（skip_clear=True）
2. 如果之前被删除，重新运行"跑图生成视频"即可
3. 建议定期备份重要的图片文件

### Q3: 日志中没有看到阶段提示？
**A**: 
1. 确认使用的是修复后的代码
2. 检查是否运行的是"跑图生成视频"任务
3. 如果是"直接生成视频"，不会有完整的阶段提示

### Q4: 图像生成耗时特别长？
**A**: 
1. 检查SD WebUI是否正常运行
2. 查看SD服务的响应时间
3. 考虑降低图片分辨率或采样步数
4. 检查网络连接是否稳定
