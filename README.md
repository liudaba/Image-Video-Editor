# 短视频生成器 - 使用说明书

## 这是什么？

这是一个**自动生成短视频**的工具。你只需要：
1. 准备一段音频（比如讲故事、解说、配音）
2. 点击几个按钮
3. 程序就会自动帮你生成一个带画面的视频

**适合谁用？**
- 想做短视频但不会剪辑的人
- 想把音频变成视频的人
- 想批量生成视频内容的人

## 第二步：安装必要的软件

### 必须安装的软件

#### 1. Python（程序的运行的基础）

**检查是否已经安装：**
- 按 `Win + R` 键，输入 `cmd`，回车
- 输入 `python --version`，回车
- 如果显示版本号（比如 Python 3.11.0），说明已安装，跳到下一步
- 如果显示"不是内部或外部命令"，说明没安装，继续看下面

**安装方法：**
1. 打开 https://www.python.org/downloads/
2. 点击 **"Download Python 3.11.x"**（最新的3.11版本）
3. 下载后双击安装
4. **重要**：安装时勾选 **"Add Python to PATH"**，然后点击 Install Now
5. 等待安装完成

#### 2. FFmpeg（处理视频必需的）

**这是什么？** 这是一个处理视频的工具，我们的程序需要它来生成视频。

**安装方法：**

**Windows用户：**
1. 打开 https://github.com/BtbN/FFmpeg-Builds/releases
2. 找到最新版本，下载 `ffmpeg-master-latest-win64-gpl.zip`
3. 解压到 `C:\ffmpeg` 文件夹
4. 设置环境变量：
   - 右键"此电脑" → 属性 → 高级系统设置 → 环境变量
   - 在"系统变量"中找到 Path，双击打开
   - 点击"新建"，输入 `C:\ffmpeg\bin`
   - 点击确定保存

**验证安装：**
- 打开cmd，输入 `ffmpeg -version`
- 如果显示版本信息，说明安装成功

## 第三步：安装Python依赖包

**什么是依赖包？** 就是程序运行需要的其他工具包。

**安装方法：**

1. 打开cmd（按 `Win + R`，输入 `cmd`，回车）
2. 进入项目文件夹：
   ```bash
   cd C:\Users\你的用户名\Desktop\短视频生成器
   ```
   （注意：根据你实际放的位置修改路径）
3. 输入以下命令，回车：
   ```bash
   pip install -r requirements.txt
   ```
4. 等待安装完成（可能需要几分钟，取决于网速）

**如果安装很慢或失败：**
- 可以尝试使用国内镜像：
  ```bash
  pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
  ```

### 第四步（可选）：安装Ollama本地大模型

**这是什么？** Ollama是一个可以在本地运行大语言模型的工具。安装后，程序可以使用本地大模型来生成更智能的分镜脚本和画面描述。

**为什么是可选的？**
- 如果不安装Ollama，程序会使用内置的规则来生成画面描述
- 安装Ollama后，可以获得更智能、更精准的生成效果

**安装方法：**

1. 访问 Ollama 官网：https://ollama.com/download
2. 下载 Windows 版本
3. 双击安装包，安装Ollama
4. 安装完成后，打开cmd，输入 `ollama --version` 验证安装

**下载模型（推荐）：**

安装Ollama后，需要下载至少一个模型。推荐下载：
- **qwen2.5**：中文能力强，性价比高
  ```bash
  ollama pull qwen2.5
  ```
- **llama3.2**：英文能力强
  ```bash
  ollama pull llama3.2
  ```
- **deepseek-r1**：推理能力强
  ```bash
  ollama pull deepseek-r1
  ```

**注意：** 模型文件较大（1-5GB），下载需要一定时间。程序第一次运行时会自动检测Ollama服务。

## 第五步：下载AI模型文件

**什么是模型文件？** 这是AI的大脑，用来识别语音、生成图片等。

**注意：** 这些文件很大（共约800MB），所以没放在Git仓库里，需要你自己下载。

### 需要下载的模型：

#### 1. Whisper模型（语音识别用）

**文件名：** `tiny.pt` 或 `medium.pt`

**下载地址：**
- 程序第一次运行时会自动下载（推荐）
- 或者手动下载：https://huggingface.co/openai/whisper-medium

**放哪里：** 下载后放到 `models/` 文件夹里

#### 2. LatentSync模型（口型同步用，可选）

**文件名：** `latentsync_unet.pt`

**说明：** 这个模型用于让视频人物的口型和音频对齐。如果你只是做图片视频，不需要这个。

**下载地址：** 需要从其他渠道获取（文件太大，约700MB）

**放哪里：** 下载后放到 `models/` 文件夹里

## 第七步：使用教程

### 基本操作流程

#### 1. 导入音频

- 点击界面上的 **"导入音频"** 按钮
- 选择你的音频文件（支持 mp3, wav, ogg 格式）
- 等待音频加载完成

#### 2. 生成分镜脚本

- 点击 **"一键生成分镜"** 按钮
- 程序会自动：
  1. 识别音频中的语音内容
  2. 根据内容生成画面描述
  3. 计算每个画面显示的时间
- 等待生成完成（可能需要几分钟）
- 生成的分镜脚本会显示在界面上

#### 3. 生成图片

**方法A：用AI自动生成（需要配置Stable Diffusion）**
- 在界面上配置 Stable Diffusion API 地址
- 点击生成图片按钮
- 等待AI生成图片

**方法B：自己准备图片（推荐新手用）**
- 打开项目文件夹里的 `images/` 目录（如果没有就新建一个）
- 把你准备好的图片放进去
- 图片命名格式：`001.png`, `002.png`, `003.png`...（对应第1个、第2个、第3个分镜）

#### 4. 生成视频

- 点击 **"生成视频"** 按钮
- 程序会把图片和音频合成为一个视频
- 生成的视频会保存在 `output_project/` 文件夹里

### 新功能：Ollama本地大模型

#### 什么是Ollama？

Ollama是一个在本地运行大语言模型的工具。安装后，程序可以使用本地AI来生成更智能的分镜脚本。

#### 如何使用？

1. **确保Ollama已安装并运行**
   - 首次运行时，程序会自动检测Ollama服务
   - 如果未安装，请参考上面的"安装Ollama"说明

2. **选择模型**
   - 在左侧面板找到"Ollama模型"选项
   - 点击选择你已下载的模型（如 qwen2.5、llama3.2 等）

3. **使用本地大模型生成分镜**
   - 选择模型后，点击"一键生成分镜"
   - 程序会使用本地大模型分析音频内容
   - 自动生成更精准的画面描述

#### 推荐配置

- **qwen2.5**：推荐用于中文内容，性价比高
- **llama3.2**：适合英文内容
- **deepseek-r1**：适合需要深度推理的内容

### 新功能：多模型融合系统

#### 什么是多模型融合？

多模型融合系统可以同时使用多个AI模型来生成画面描述，然后智能合并各模型的优势，生成更高质量的提示词。

#### 如何启用？

1. 在高级设置中启用"多模型融合"
2. 选择参与融合的模型（至少2个）
3. 设置融合策略：
   - **取长补短**：各模型取最优部分组合
   - **投票机制**：多数模型认同的内容优先
   - **加权平均**：根据模型表现动态调整权重

### 重要设置项

#### 过渡模式（在设置区选择）

- **硬切**（推荐）：直接切换画面，速度最快
- **淡入淡出**：画面慢慢出现/消失，较慢
- **交叉溶解**：两个画面叠加过渡，较慢

**建议：** 如果追求速度，选"硬切"；如果想要好看的效果，选"淡入淡出"

#### 分辨率设置

- **默认**：1920x1080（1080P高清）
- **可选**：根据你的需要调整
- **建议**：保持默认即可

## 常见问题解答

### Q1: 程序打不开怎么办？

**可能原因和解决方法：**

1. **提示"找不到Python"**
   - 原因：Python没安装或没添加到PATH
   - 解决：重新安装Python，记得勾选"Add Python to PATH"

2. **提示"缺少xxx模块"**
   - 原因：依赖包没安装完整
   - 解决：重新运行 `pip install -r requirements.txt`

3. **双击没反应**
   - 原因：可能是Python路径问题
   - 解决：尝试用cmd命令行运行，看有什么错误提示

### Q2: 生成视频很慢怎么办？

**加速方法：**

1. **使用"硬切"过渡模式**（默认就是）
2. **关闭动画效果**（在设置里选"无"）
3. **使用GPU加速**（如果有NVIDIA显卡）
4. **减少分镜数量**（每个分镜都会生成一张图，图越多越慢）

### Q3: 音画不同步怎么办？

**检查以下几点：**

1. 确保使用的是最新版本代码
2. 分镜生成时音频要完整识别
3. 图片数量要和分镜数量一致
4. 如果还是不同步，尝试重新生成分镜

### Q4: 生成的图片质量不好怎么办？

**改善方法：**

1. **使用Stable Diffusion本地部署**：自己搭建SD，画质更好控制
2. **调整提示词**：在分镜脚本里修改画面描述，越详细越好
3. **使用更好的模型**：比如用SDXL代替SD 1.5

### Q5: 可以批量生成吗？

**目前版本：** 一次只能处理一个音频文件

**变通方法：**
- 准备多个音频文件
- 一个一个导入生成
- 或者同时打开多个程序窗口（如果电脑配置够好）

### Q6: 模型文件下载太慢/失败怎么办？

**解决方法：**

1. **Whisper模型**：让程序自动下载，第一次用时会自动下载
2. **LatentSync模型**：
   - 可以从百度网盘等国内镜像下载
   - 或者暂时不用口型同步功能（不影响主要功能）

### Q7: 如何使用Ollama本地大模型？

**使用方法：**

1. **安装Ollama**：参考上面的安装说明
2. **下载模型**：在cmd中运行 `ollama pull qwen2.5`（或其他模型）
3. **启动程序**：程序会自动检测Ollama服务
4. **选择模型**：在左侧面板点击"Ollama模型"按钮，选择要使用的模型
5. **生成分镜**：点击"一键生成分镜"，程序会使用本地大模型生成画面描述

### Q8: SD提示词和豆包提示词有什么区别？

**SD提示词（英文）：**
- 输出的提示词是英文
- 适合配合英文的Stable Diffusion模型使用
- 关键词更简洁，便于SD模型理解

**豆包提示词（中文）：**
- 输出的提示词是中文
- 语义更丰富，描述更详细
- 适合中文用户理解和编辑

**建议：** 如果你使用秋叶的Stable Diffusion WebUI，推荐使用"豆包提示词"；如果使用英文版SD，推荐使用"SD提示词"。

### Q9: 如何选择视觉风格？

**使用方法：**

1. 点击 **"高级设置"** 按钮
2. 在 **"风格设置"** 部分展开风格选项
3. 勾选你想要的风格（可多选）
4. 点击 **"应用设置"**

**风格选择建议：**

- **纪录片/新闻类内容**：选择"纪录片风"、"写实摄影"
- **科技/未来类内容**：选择"赛博朋克"、"电影感"
- **故事/动画类内容**：选择"皮克斯"、"吉卜力"、"日式动漫"
- **艺术/创意类内容**：选择"油画"、"达芬奇"、"水彩"、"梵高"
