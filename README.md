## 这个工具是干什么的？

简单说：**你给它一段音频（比如一段朗读、一段解说），它就能自动生成一个带画面的短视频。**

它是怎么做到的？分4步：

```
音频 → 语音识别（把声音变成文字）→ 分镜脚本（把文字切成一个个场景）→ AI画图（给每个场景画一张图）→ 合成视频（图片+音频=视频）
```

**适合做什么类型的视频？**
- 📚 知识科普（比如"宇宙是怎么来的"）
- 📖 有声书可视化（给小说配画面）
- 🎓 教育课程（给讲解配图）
- 📰 新闻资讯配音视频
- 🎵 音乐MV

## 使用前需要准备什么

### 硬件要求

| | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | Intel i5 / AMD Ryzen 5 | Intel i7 / AMD Ryzen 7 |
| 内存 | 8GB | 16GB |
| 显卡 | NVIDIA GTX 1060（4GB显存） | NVIDIA RTX 3060（8GB显存） |
| 硬盘 | 10GB可用空间 | SSD固态硬盘 |

### 必须安装的软件

你的电脑上需要提前装好这3个东西：

| 软件 | 干什么用的 | 怎么安装 |
|------|-----------|----------|
| **Python 3.10+** | 运行本程序 | 去 python.org 下载安装 |
| **Ollama** | AI大模型，用来分析内容和生成提示词 | 去 ollama.com 下载安装，装好后至少下载一个模型（推荐 `gemma3:4b` 或 `qwen3:8b`） |
| **Stable Diffusion WebUI** | AI画图，用来生成场景图片 | 搜索"Stable Diffusion WebUI 安装教程"，启动时需要加 `--api` 参数 |

> 💡 **简单理解**：Ollama 是"大脑"（负责想），Stable Diffusion 是"画笔"（负责画）

### 首次运行

1. 双击 **`检查环境.bat`**，检查Python和依赖是否安装完整
2. 如果提示缺少依赖包，在命令行运行：`pip install -r requirements.txt`
3. 确保Ollama和Stable Diffusion WebUI都在运行
4. 双击 **`start.bat`** 启动程序

## 快速开始（3步出视频）

### 第1步：启动程序

双击 **`start.bat`**

等待约30秒，当日志区域显示 **"✅ 程序启动完成，工具已就绪！"** 就可以开始了

> 💡 启动时程序会自动检测Ollama和SD服务，如果Ollama没启动会弹窗提醒你

### 第2步：导入音频

点击左侧 **"📂 导入音频"** 按钮，选择你的音频文件

支持格式：`.mp3` `.wav` `.m4a` `.flac`，文件大小不超过500MB

### 第3步：生成视频

点击左侧 **"🎞️ 生成视频"** 按钮，程序会自动完成所有工作：

1. 分析音频内容 → 2. 生成分镜脚本 → 3. 生成场景图片 → 4. 合成视频

⏱️ 大约需要10-20分钟（取决于音频长度和电脑配置）

完成后视频保存在 `output_project` 文件夹中，点击 **"📁 打开文件夹"** 即可查看

## 详细操作步骤

### 方式一：一键模式（推荐新手）

**最简单的用法，只需要2次点击：**

```
导入音频 → 点击"🎞️ 生成视频" → 等待完成
```

程序会自动判断当前状态，智能执行：
- 如果没有分镜脚本 → 先生成分镜，再画图，再合成
- 如果有分镜但没有图片 → 先画图，再合成
- 如果分镜和图片都有 → 直接合成视频

### 方式二：分步模式（推荐进阶用户）

**想要更多控制力？可以分步操作：**

#### 第1步：生成分镜脚本

点击 **"🎬 生成分镜脚本"**

程序会自动完成4件事：

1. **语音识别** — 用Whisper把音频转成文字（带时间戳）
2. **内容分析** — 用Ollama大模型分析文章主题、情感、视觉基调
3. **语音纠错** — 自动纠正语音识别中的常见错误（如同音字、专有名词）
4. **生成提示词** — 为每个场景生成AI画图的提示词（两阶段思考：先理解语义，再翻译为视觉场景）

> 💡 分镜脚本保存在 `output_project/shots_data.json`

#### 第2步：生成视频

点击 **"🎞️ 生成视频"**

程序会自动完成：
1. 连接SD API生成场景图片
2. 把图片和音频合成为视频

> 💡 生成的图片在 `output_project/images/` 文件夹，视频在 `output_project/` 文件夹

### 关于垃圾桶功能 🗑️

程序有一个**安全垃圾桶**机制，防止误删文件：

- 每次清理数据时，旧文件不会直接删除，而是移到"垃圾桶"文件夹
- 垃圾桶按时间分类（如 `清理_20260427_203000`），方便找回
- 如果误删了重要文件，可以从垃圾桶里手动复制回来
- 垃圾桶不会自动清空，需要手动清理

**配置模式说明：**

| 模式 | 特点 | 适合场景 |
|------|------|----------|
| 创意模式 | 创造性高，结果多样 | 头脑风暴、创意内容 |
| 平衡模式 | 创造性和准确性兼顾 | 一般用途 |
| 精确模式 | 准确性高，结果稳定 | 严谨内容、科普 |
| 极速模式 | 响应最快 | 简单任务、快速测试 |
| **质量优先**（默认） | 输出质量最高 | 大多数场景 |

### ⚡ 并发设置

| 设置项 | 选项 | 说明 |
|--------|------|------|
| 分镜创建线程数 | 4/8/12/**16**/24/32 | 同时画图的线程数，越大越快但越占显存 |
| 提示词生成线程数 | 1/2/3/**4**/6/8 | 同时生成提示词的线程数 |

> ⚡ **提示**：如果显存不够（生成图片时报错），把分镜创建线程数调低到4或8

| 类型 | 说明 |

| **SD提示词**（默认） | 标准Stable Diffusion提示词，支持风格预设叠加，自动适配不同制图模型 |
| ARV写实提示词 | 绝对写实风格提示词，追求照片级真实感，选择后风格选项自动禁用 |

> 💡 选择"SD提示词"时，程序会根据你选择的制图模型自动使用对应的提示词模板（SD1.5/SDXL/Flux/SD3），无需手动切换

## 常见问题

### Q1：程序启动后弹窗说"Ollama服务未连接"？

**原因**：Ollama没有运行

**解决方法**：
1. 打开Ollama应用程序（在开始菜单搜索"Ollama"）
2. 等待Ollama启动完成（任务栏出现羊驼图标）
3. 重新启动本程序

> 💡 确保Ollama中至少安装了一个模型。在命令行运行 `ollama list` 查看已安装的模型，如果没有，运行 `ollama pull gemma3:4b` 下载

### Q2：生成图片时一直卡住不动？

**可能原因和解决方法**：

| 原因 | 解决方法 |
|------|----------|
| SD WebUI没启动 | 启动Stable Diffusion WebUI（记得加 `--api` 参数） |
| SD API地址不对 | 在高级设置中检查API地址是否为 `http://127.0.0.1:8080` |
| 显存不足 | 关闭浏览器等占显存的程序，或降低图片分辨率 |
| SD正在处理中 | 耐心等待，SD生成一张图需要几秒到十几秒 |

### Q3：生成的图片和音频内容对不上？

**解决方法**：
1. 在高级设置的"主题自定义"中，手动填写核心主题和视觉基调
2. 尝试切换提示词类型（SD提示词 ↔ ARV写实提示词）
3. 尝试更换Ollama模型（推荐 `gemma3:4b` 或 `qwen3:8b`）
4. 确认制图模型选择正确——下拉菜单中的类型标签（[SD1.5]/[SDXL]/[Flux]/[SD3]）是否与实际模型匹配

### Q4：视频合成失败？

**检查清单**：
- ✅ 音频文件是否还在（不要删除导入的音频文件）
- ✅ 磁盘空间是否充足（至少留5GB）
- ✅ 图片是否全部生成完成（看日志有没有报错）

**解决方法**：点击垃圾桶🗑️清除所有数据，重新导入音频从头开始

### Q5：程序运行很慢？

**优化建议**：
- 关闭其他占用GPU的程序（浏览器、游戏等）
- 在高级设置中降低图片分辨率（如 1280x720）
- 降低分镜创建线程数（减少显存占用）
- 使用更小的Whisper模型（如 small 或 base）

### Q6：生成的视频没有声音？

**检查**：
- 原始音频文件是否正常（用播放器试听一下）
- 音频格式是否支持（.mp3 .wav .m4a .flac）
- 音频音量是否太低

### Q7：如何重新开始？

点击音频状态旁边的 **🗑️ 垃圾桶按钮**，会清除当前音频和所有相关数据，然后重新导入音频即可

> 💡 旧文件会移到垃圾桶文件夹，不会直接删除，需要的话可以从垃圾桶找回

### Q8：下载了新的SD模型，程序里看不到？

**解决方法**：
1. 确认新模型已放入SD WebUI的 `models/Stable-diffusion/` 目录
2. 确认SD WebUI正在运行
3. 在高级设置的模型下拉菜单旁，点击 **🔄 刷新按钮**
4. 下拉菜单会自动更新，新模型会显示类型标签（如 `[SDXL] 新模型名`）

> 💡 程序会自动识别模型类型（SD1.5/SDXL/Flux/SD3），并自动切换对应的提示词模板和生成参数

## 注意事项

### 使用建议

1. **一次只处理一个音频** — 不要同时运行多个程序实例
2. **音频质量很重要** — 录音越清晰，识别越准确，效果越好
3. **不要中途关闭程序** — 视频生成过程中关闭程序可能导致文件损坏
4. **定期清理垃圾桶** — 垃圾桶不会自动清空，久了会占硬盘空间
5. **保持服务运行** — 生成视频期间不要关闭Ollama和SD WebUI

### 文件说明

| 文件/文件夹 | 说明 |
|-------------|------|
| `start.bat` | 双击启动程序 |
| `检查环境.bat` | 检查Python环境是否正常 |
| `output_project/` | 输出目录，所有生成的文件都在这里 |
| `output_project/images/` | 生成的场景图片 |
| `output_project/shots_data.json` | 分镜脚本数据 |
| `config.json` | 程序配置文件（自动生成） |
| `垃圾桶/` | 被清理的旧文件存放处 |

### 性能参考

基于 NVIDIA RTX 4000（8GB显存）的测试结果：

| 任务 | 大约耗时 | 说明 |
|------|----------|------|
| 语音识别 | ~30秒 | Whisper medium模型 |
| 内容分析 | ~6秒 | Ollama大模型 |
| 提示词生成 | ~70秒 | 4线程并行 |
| 图片生成 | ~8秒/张 | SD模型 |
| 视频合成 | ~30秒 | MoviePy编码 |

> 💡 3分钟音频大约产生40-60个分镜，全流程约10-20分钟
