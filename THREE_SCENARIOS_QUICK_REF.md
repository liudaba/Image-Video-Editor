## 📋 三种情况详解

### 情况1: 有分镜脚本 + 有图片（数量匹配）✅

**触发条件**: 
- 文件夹有 `shots_data.json`
- images 文件夹有图片
- 图片数量 == 分镜数量

**执行流程**:
```
直接使用现有文件 → 合成视频 → 渲染视频
```

**日志提示**:
```
============================================================
✅ 检测到完整的分镜脚本和图片文件
============================================================
   📋 分镜数量: 94 个
   🖼️ 图片数量: 94 张
   ✅ 数量匹配，可以直接合成视频

💡 提示: 将直接使用现有文件，跳过生成分镜和生成图片步骤

🎞️ 开始生成视频...
✅ 所有图片已存在，跳过生成步骤
🎬 所有图片已就绪，开始视频合成...
✅ 视频生成完成！
```

**耗时**: 2-3分钟

---

### 情况2: 有分镜脚本 + 无图片/图片不匹配 ✅

**触发条件**: 
- 文件夹有 `shots_data.json`
- images 文件夹为空 或 图片数量 != 分镜数量

**子情况2a: 无图片**
```
============================================================
✅ 检测到分镜脚本文件，但图片文件夹为空
============================================================
   📋 分镜数量: 94 个
   🖼️ 图片数量: 0 张

💡 提示: 将使用现有分镜脚本，自动生成图片
```

**子情况2b: 图片数量不匹配**
```
============================================================
⚠️ 检测到分镜脚本文件，但图片数量不匹配
============================================================
   📋 分镜数量: 94 个
   🖼️ 图片数量: 50 张
   ❌ 数量不匹配（需要 94 张）

💡 提示: 将使用现有分镜脚本，重新生成所有图片
```

**执行流程**:
```
阶段1: 准备分镜数据
  ├─ 加载现有分镜脚本
  └─ 日志: "ℹ️ 将直接使用文件夹内分镜脚本生成图片"

阶段2: 生成图像
  ├─ 检测图片缺失
  ├─ 调用 generate_images()
  └─ 日志: "✅ 图像生成完成 (耗时: XX.Xs)"

阶段3: 视频合成
  └─ 自动继续
```

**耗时**: 8-12分钟

---

### 情况3: 无分镜脚本 ✅

**触发条件**: 
- 文件夹没有 `shots_data.json`

**日志提示**:
```
============================================================
📝 未检测到分镜脚本文件
============================================================

💡 提示: 将从头开始生成分镜脚本、图片和视频
```

**执行流程**:
```
阶段1: 准备分镜数据
  ├─ 清除上一次任务的缓存
  │   ├─ 清空 self.shots_data
  │   ├─ 删除 _pregenerated_prompts
  │   ├─ 删除旧的 shots_data.json
  │   ├─ 清除音频分析缓存
  │   ├─ 清除 prompt_cache
  │   └─ 清除 image_cache
  │
  ├─ 日志: "✅ 旧数据已清除，开始生成分镜..."
  └─ 调用 generate_shots(auto_mode=True)
      ├─ 转录音频 (Whisper)
      ├─ 分析内容 (Ollama)
      └─ 生成分镜脚本

阶段2: 生成图像
  ├─ 检测图片全部缺失
  ├─ 调用 generate_images()
  └─ 日志: "✅ 图像生成完成 (耗时: XX.Xs)"

阶段3: 视频合成
  └─ 自动继续
```

**耗时**: 15-20分钟

## 🔍 关键功能说明

### 1. 自动清除缓存

**何时清除**: 仅在情况3（从头生成）时清除

**清除内容**:
- ✅ 内存中的分镜数据 (`self.shots_data`)
- ✅ 预生成的提示词缓存 (`_pregenerated_prompts`)
- ✅ 旧的分镜脚本文件 (`shots_data.json`)
- ✅ 音频分析缓存
- ✅ 提示词缓存 (`prompt_cache`)
- ✅ 图像缓存 (`image_cache`)
- ✅ 状态管理器

**日志提示**:

