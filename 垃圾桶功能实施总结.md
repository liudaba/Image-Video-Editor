def _cleanup_residual_files(self):
    """启动时清理上次可能残留的磁盘文件，移动到垃圾桶而非直接删除"""
    try:
        # 1. 创建垃圾桶文件夹（如果不存在）
        trash_dir = os.path.join(os.path.dirname(os.path.abspath(__file__)), "垃圾桶")
        if not os.path.exists(trash_dir):
            os.makedirs(trash_dir)
        
        # 2. 为本次清理创建带时间戳的子文件夹
        import datetime
        timestamp = datetime.datetime.now().strftime("%Y%m%d_%H%M%S")
        trash_session_dir = os.path.join(trash_dir, f"清理_{timestamp}")
        os.makedirs(trash_session_dir)
        
        # 3. 移动文件到垃圾桶（而非删除）
        #    - shots_data.json
        #    - images/*.png, *.jpg
        #    - *.mp4, *.avi
        
        # 4. 记录日志
        self.log(f"🗑️ 已将 {moved_count} 个文件移至垃圾桶: {trash_session_dir}")

## 🎯 功能特点

### ✅ 优势

1. **防止误删**: 文件不会立即永久删除，给用户后悔的机会
2. **按时间分类**: 每次清理创建独立的时间戳文件夹，便于追溯
3. **结构清晰**: 
   ```
   垃圾桶/
   ├── 清理_20260427_203000/
   │   ├── shots_data.json
   │   ├── images/
   │   │   ├── shot_001.png
   │   │   └── ...
   │   └── video.mp4
   ├── 清理_20260428_150000/
   └── ...
   ```
4. **日志提示**: 清理完成后显示移动的文件数量和路径
5. **异常处理**: 单个文件移动失败不影响其他文件

### ⚠️ 注意事项

1. **空间占用**: 垃圾桶会随时间增长，建议定期手动清理
2. **不自动清空**: 需要用户手动管理垃圾桶内容
3. **首次创建**: 第一次运行时会创建垃圾桶文件夹并显示提示

## 🔍 技术细节

### 使用的Python模块
- `os` - 文件和目录操作
- `datetime` - 时间戳生成
- `shutil` - 文件移动（在测试中使用）

### 关键API
- `os.path.join()` - 路径拼接
- `os.makedirs()` - 创建目录
- `os.rename()` / `shutil.move()` - 移动文件
- `os.listdir()` - 列出目录内容

### 异常处理
- 单个文件移动失败不会中断整个流程
- 所有异常都会记录到日志中
- 使用 `try-except` 保护每个文件操作
